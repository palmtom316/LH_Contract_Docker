## 1. 数据持久化验证
根据 `docker-compose.prod.yml` 的配置分析：
- **数据库 (PostgreSQL)**: 使用了具名卷 `postgres_data`。
    - **安全性**: Docker 具名卷通常位于 `/var/lib/docker/volumes/`，不会因容器删除而丢失。
    - **风险点**: 不要执行 `docker-compose down -v` (带 `-v` 标志)，否则会删除数据卷！
- **上传文件 (Uploads)**: 映射路径为 `${HOST_UPLOAD_DIR:-./uploads}:/app/uploads`。
    - **安全性**: 这是一个主机绑定挂载（Bind Mount）。数据直接存储在宿主机目录。
    - **风险点**: 确保 `${HOST_UPLOAD_DIR}` 环境变量在升级前后保持一致，否则新容器会指向错误的目录。

## 2. 推荐的“双保险”升级流程

### 步骤 1: 应用级冷备份 (Secondary Backup)
虽然已有 VM 快照，但建议进行一次**应用级数据导出**，以便在无需回滚整个 VM 的情况下快速恢复数据。

```bash
# 1. 创建备份目录
mkdir -p ./upgrade_backups/$(date +%Y%m%d)

# 2. 导出数据库 (无需停止服务)
# 注意：使用当前运行的容器名
docker exec lh_contract_db_prod pg_dump -U lh_admin lh_contract_db > ./upgrade_backups/$(date +%Y%m%d)/db_full_backup.sql

# 3. 备份上传文件 (保留权限)
# 假设上传目录在当前目录的 uploads/
tar -czvf ./upgrade_backups/$(date +%Y%m%d)/uploads_backup.tar.gz ./uploads
```

### 步骤 2: 平滑停机
```bash
# 停止容器，但保留网络和卷
docker-compose -f docker-compose.prod.yml stop
```

## 3. 回滚方案 (Rollback Strategy)

如果在升级后发现重大 BUG：

### 方案 A: 代码回滚 (推荐，快速)
1. 切换回旧版本代码: `git checkout <previous_tag>`
2. 重建并重启: `docker-compose -f docker-compose.prod.yml up -d --build`
3. **注意**: 如果涉及了数据库 schema 变更，且无法向下兼容，则必须使用方案 B 或恢复数据库备份。

### 方案 B: 数据库恢复 (数据回滚)
如果新版本破坏了数据结构：
1. 停止服务: `docker-compose -f docker-compose.prod.yml down`
2. 启动空数据库:
   ```bash
   docker volume rm lh_contract_docker_postgres_data # ⚠️ 警告：这将删除当前脏数据
   docker-compose -f docker-compose.prod.yml up -d db
   ```
3. 导入备份:
   ```bash
   cat ./upgrade_backups/<date>/db_full_backup.sql | docker exec -i lh_contract_db_prod psql -U lh_admin -d lh_contract_db
   ```

### 方案 C: VM 快照恢复 (终极手段)
如果上述方法均失败，直接在云服务商控制台恢复 VM Snapshot。

## 4. 特别安全检查项

1. **`init.sql` 安全性**:
   - 您的 `init.sql` 使用了 `CREATE EXTENSION IF NOT EXISTS`，这是安全的（幂等操作）。
   - **风险**: 不要修改 `init.sql` 来包含数据修改逻辑，因为它只在**首次**创建数据卷时运行。升级时的变更必须通过迁移脚本或外部命令执行。

2. **文件权限**:
   - 升级过程中，确保 `uploads` 目录的属主（Owner）保持不变。Docker 容器内的用户 ID (通常是 root 或 1000) 必须有权读写宿主机的 `uploads` 目录。
   - 建议在启动后运行：`docker exec lh_contract_backend_prod ls -ld /app/uploads` 检查可写性。
