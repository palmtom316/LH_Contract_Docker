## 常见问题排查

### 问题 1: 文件下载 404

**症状**: PDF/文件链接无法访问

**排查步骤**:

```bash
# 1. 检查 Nginx 代理配置
docker exec lh_contract_nginx cat /etc/nginx/conf.d/default.conf | grep -A 10 "/uploads/"

# 2. 检查后端静态文件挂载
docker exec lh_contract_backend_prod ls -la /app/uploads/

# 3. 检查宿主机目录
ls -la ${HOST_UPLOAD_DIR:-./uploads}
```

**解决**: 确保 `docker-compose.prod.yml` 中的挂载路径正确：

```yaml
volumes:
  - ${HOST_UPLOAD_DIR:-./uploads}:/app/uploads
```

### 问题 2: 数据库连接失败

**症状**: `password authentication failed for user "lh_admin"`

**排查步骤**:

```bash
# 1. 检查 .env.production 中的密码
cat .env.production | grep POSTGRES_PASSWORD

# 2. 检查容器内的环境变量
docker-compose -f docker-compose.prod.yml exec backend env | grep DATABASE_URL
```

**解决**: 确保 `.env.production` 中的密码与数据库初始化时一致。

### 问题 3: SECRET_KEY 错误

**症状**: 应用启动失败，日志显示 `SECRET_KEY 环境变量在生产环境中必须设置!`

**解决**:

```bash
# 生成新密钥
python -c "import secrets; print(secrets.token_urlsafe(64))"

# 添加到 .env.production
echo "SECRET_KEY=<生成的密钥>" >> .env.production
```
