# V1.5.1 to V1.5.3 Upgrade Guide

## Overview
This guide documents the process for upgrading the **LH Contract Management System** from version **1.5.1** to **1.5.3** on a VM environment.

**Version 1.5.3 Changes:**
- **UI Optimization:** Adjusted table column widths and formatting for Upstream, Downstream, Management, and Expense lists.
- **Display Fixes:** Money amounts and dates are now forced to display on a single line.
- **Zoom Reset:** Removed the experimental 80% zoom scaling, restoring the standard 100% display scale.
- **Footer/Summary Rows:** Prevented text wrapping in table summary rows.

## Prerequisites
- Existing installation of V1.5.1 running via Docker Compose.
- Access to the server terminal.
- Git access to the repository.

## Upgrade Steps

### 1. Pull Latest Code
Navigate to your project directory and fetch the latest updates from GitHub.

```bash
# Enter project directory
cd /home/palmtom/projects/LH_Contract_Docker  # Adjust path if different

# Fetch latest branches
git fetch origin

# Checkout the new version branch
git checkout 1.5.3

# Pull the latest changes
git pull origin 1.5.3
```

### 2. Rebuild Services
Since V1.5.3 now includes backend fixes for search filters, you need to rebuild both backend and frontend services.

**For Standard/Development Environment:**
```bash
docker-compose build backend frontend
docker-compose up -d backend frontend
```

**For Production Environment:**
(If you run with `-f docker-compose.prod.yml`)
```bash
docker-compose -f docker-compose.prod.yml build backend frontend
docker-compose -f docker-compose.prod.yml up -d backend frontend
```

### 3. Verification
1.  **Clear Cache:** Open your browser and navigate to the application. Perform a **Hard Refresh** (Ctrl+F5 on Windows/Linux, Cmd+Shift+R on macOS) to clear cached JavaScript/CSS files.
2.  **Check Zoom:** Verify that the pages are no longer shrunk to 80% scale.
3.  **Check Tables:** Go to **Upstream Contract List**:
    - Verify that "Contract Amount", "Receivable", etc., are displayed on a single line.
    - Scroll to the bottom and check that the "Total" (Summary) row text does not wrap.
4.  **Check Other Pages:** Verify similar behavior on **Downstream**, **Management**, and **Expense** pages.

## Troubleshooting

**Issue: Changes not visible after upgrade.**
- **Solution:** Browser cache is the most common cause. Try opening the page in Incognito/Private mode. If it works there, clear your main browser's cache for the site.
- **Solution:** Ensure the docker build command completed successfully and the container was recreated (`Up` status).

**Issue: "1.5.3 branch not found"**
- **Solution:** Run `git fetch origin` to ensure your local git knows about the new remote branch.
