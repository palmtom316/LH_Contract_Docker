# 日期输入筛选与筛选栏重排设计

## 概述

本次改造覆盖前端所有使用“起始日期/结束日期”进行筛选的模块，包括合同列表、费用列表、审计日志、报表导出卡片以及合同查询机器人。目标是彻底移除筛选场景中的日期选择器，统一改为用户手动输入日期，并由系统对常见模糊格式进行识别、校验与格式化。

改造同时包含各页面筛选栏的结构整理。现有页面中日期筛选控件宽度不一致、占位过宽、换行策略混乱。本次会建立统一的筛选栏排布规则，使桌面端和窄屏下都能保持清晰、紧凑且稳定的显示状态。

## 目标

- 所有起始日期/结束日期筛选统一改为纯文本输入。
- 支持用户输入多种常见日期格式，并统一转换为 `YYYY/MM/DD` 展示格式。
- 无法识别的输入保留原文，向用户提示错误，并且该字段不参与筛选。
- 开始日期晚于结束日期时，范围整体标记为无效，不执行该范围筛选。
- 筛选栏布局统一，减少各页面重复实现和视觉漂移。

## 非目标

- 不改动表单新增/编辑弹窗中的业务日期录入方式，除非该字段本身已经明确使用 `SmartDateInput`。
- 不在本次范围内引入自然语言日期解析，例如“昨天”“上个月”“本周一”。
- 不支持无分隔符的紧凑格式如 `20260406`，除非现有组件中已有依赖且不影响主规则。

## 影响范围

### 页面范围

- 合同管理列表
- 上游合同列表
- 下游合同列表
- 普通费用列表
- 零工费用列表
- 审计日志页面
- 报表导出页中的所有日期范围筛选卡片
- 合同查询机器人中的签约日期范围

### 组件范围

- `frontend/src/components/SmartDateInput.vue`
- 新增统一的日期范围输入组件
- `frontend/src/components/ui/AppFilterBar.vue`
- `frontend/src/components/ui/AppRangeField.vue` 或其替代实现

## 当前问题

1. 多个页面直接使用 `el-date-picker type="daterange"`，与“纯输入日期”的目标冲突。
2. 现有 `SmartDateInput` 仅支持有限格式，且默认输出为 `YYYY-MM-DD`，与本次目标格式不一致。
3. 各列表页筛选栏布局分散实现，日期范围控件宽度过大，和关键词、状态、操作按钮之间的视觉节奏不统一。
4. 报表页卡片内的日期范围控件过宽，导致卡片内信息与操作区域的平衡较差。

## 设计方案

### 1. 单日期输入组件统一化

基于现有 `SmartDateInput` 进行增强，使其成为所有筛选日期输入的基础能力。

职责：

- 接收外部规范值。
- 维护用户当前输入的原始文本。
- 在失焦与回车时触发解析。
- 解析成功后将显示值格式化为 `YYYY/MM/DD`。
- 解析失败时保留原始文本、显示错误态并提供错误提示。
- 仅在解析成功时向外发出规范日期值。

对外建议接口：

- `v-model` 绑定规范值，格式为 `YYYY-MM-DD`，便于接口请求沿用现有后端约定。
- `display-format` 内部固定显示为 `YYYY/MM/DD`，避免页面各自控制。
- `placeholder` 默认可分别传入“开始日期”“结束日期”。
- `invalid` 与 `error-message` 由组件内部维护并对外透出，供范围组件或页面容器使用。

内部策略：

- 组件内部区分“显示文本”和“有效值”。
- 用户输入不应被立即清洗，避免打断输入。
- 解析时成功则同步格式化显示文本；失败则维持原文本。

### 2. 日期模糊解析规则

允许格式：

- `2026/04/06`
- `26/4/6`
- `26.4.6`
- `2026.04.06`
- `26。4。6`
- `2026-4-6`
- `26-4-6`

归一化规则：

- 将 `.`, `。`, `-` 全部归一为 `/`。
- 连续空格去除，首尾空白去除。
- 月、日允许一位或两位数字。
- 年份允许两位或四位数字。

年份规则：

- 四位年份按原值处理。
- 两位年份统一映射到 `2000-2099`，例如 `26/4/6` 解析为 `2026/04/06`。

合法性校验：

- 必须严格是三段数字，且段位分别为年、月、日。
- 月份范围 `1-12`。
- 日期必须是该月真实存在的日期。
- 解析结果必须通过显式年月日比对校验，不能依赖浏览器宽松的 `Date` 自动进位。

标准化输出：

- 组件显示值：`YYYY/MM/DD`
- 组件对外值：`YYYY-MM-DD`

错误处理：

- 无法识别时保留原输入。
- 输入框显示错误态。
- 提示文案使用明确表达，例如“日期格式无法识别，请输入如 2026/04/06”。
- 不向外提交新的有效值。

### 3. 日期范围输入组件

新增统一范围组件，封装两个日期输入框与范围校验逻辑，用于替代所有筛选场景下的 `daterange`。

职责：

- 管理开始日期输入与结束日期输入。
- 聚合单字段错误状态。
- 在两端均有效时判断开始日期是否晚于结束日期。
- 只向外发出有效的范围值。
- 对筛选栏暴露紧凑、可换行的结构。

对外接口建议：

- `v-model:start`
- `v-model:end`
- 或单个对象形式 `{ start, end }`

推荐采用独立起止值形式，以最小化现有页面改动。

范围规则：

- 任一侧无效时，保留另一侧有效值，仅无效侧不参与筛选。
- 当两侧都有效但开始日期晚于结束日期时，范围组件标记错误，并阻止该范围整体参与筛选。
- 页面发起查询前，以组件返回的有效范围值为准，不自行再做字符串判断。

### 4. 页面接入策略

#### 列表页

合同、费用、审计日志等列表页统一将日期筛选替换为两个文本输入框：

- 开始日期
- 结束日期

保留现有关键词、状态、字典筛选与查询/重置按钮的业务逻辑，仅替换控件与布局接线。

#### 报表导出页

报表卡片中所有 `daterange` 和 `daterange-with-status` 类型卡片统一切换到新范围组件。

状态与日期范围并排时，遵循如下优先级：

- 日期范围优先占据主区域
- 状态选择器使用较窄固定宽度
- 导出按钮落在操作区，不与日期输入抢占空间

#### 合同查询机器人

签约日期范围筛选改为开始/结束两个手输输入框，仍保留现有查询条件说明文案，但应同步说明支持多种日期输入格式。

### 5. 筛选栏布局重排

统一布局原则：

- 主筛选项优先横向排列。
- 日期范围视为一个完整筛选组，不拆散到操作按钮两侧。
- 操作按钮区域与输入区域分组展示。
- 桌面端尽量保持一行完成主要操作，空间不足时优雅换行。
- 窄屏下输入区先换行，按钮区独立成行或保持末尾对齐。

建议结构：

- 第一优先级：关键词、日期范围、状态
- 第二优先级：扩展筛选项
- 操作区：查询、重置、导出、新增等按钮

样式方向：

- 单日期输入宽度显著小于旧 `daterange` 控件，减少筛选栏横向挤压。
- 日期范围两个输入框之间使用短连接符或弱分隔视觉。
- 页面间统一字段高度、最小宽度、间距与换行逻辑。
- 报表卡片中的筛选区使用更紧凑的内部间距，避免卡片头、说明、筛选、操作的节奏失衡。

## 数据流与行为

1. 用户在开始日期或结束日期输入框中输入原始文本。
2. 在失焦或回车时执行解析。
3. 若单字段解析成功，组件更新显示文本为 `YYYY/MM/DD`，并向页面写入规范值 `YYYY-MM-DD`。
4. 若单字段解析失败，保留原文并标记错误，该字段不写入新的有效值。
5. 若范围两端都有效，再检查起止顺序；顺序错误则范围整体失效。
6. 页面执行筛选、导出或查询时，仅使用组件输出的有效值。

## 测试策略

### 单元测试

新增日期解析函数测试，至少覆盖：

- 四位年正常输入
- 两位年正常输入
- 不同分隔符输入
- 中文句号分隔输入
- 一位月日输入
- 非法日期输入
- 开始日期晚于结束日期
- 空值与清空逻辑

### 组件测试

- 输入合法值后自动格式化显示
- 输入非法值后保留原始文本并显示错误
- 回车与失焦行为一致
- 范围组件在部分有效、部分无效时的输出行为

### 页面回归

- 合同列表筛选可正常请求与重置
- 费用列表筛选可正常请求与重置
- 审计日志筛选可正常请求与重置
- 报表卡片导出参数正确
- 合同查询机器人能正确传递签约日期范围
- 桌面端与窄屏布局无明显错位、压缩或换行异常

## 风险与控制

### 风险 1：页面接线分散，容易遗漏

控制：

- 通过全局搜索 `start-placeholder`, `end-placeholder`, `daterange`, `开始日期`, `结束日期` 完整梳理替换点。

### 风险 2：显示格式与接口格式不一致

控制：

- 组件内部明确区分显示文本与提交值，页面只拿提交值。

### 风险 3：解析失败状态影响已有筛选值

控制：

- 失败时不覆盖上一次有效值，避免无意清空已生效筛选。

### 风险 4：布局统一后个别页面空间不足

控制：

- 优先在通用筛选栏组件中建立响应式规则，再针对报表卡片等特殊容器做局部收敛样式。

## 实施步骤

1. 抽离并测试通用日期解析函数。
2. 增强 `SmartDateInput`，支持显示文本、错误态、规范输出。
3. 新增统一日期范围输入组件。
4. 更新通用筛选栏组件与范围字段组件。
5. 逐页替换列表页、报表页、合同查询机器人中的日期范围控件。
6. 调整筛选栏样式，完成桌面端与窄屏布局收敛。
7. 补充测试并执行前端回归验证。

## 验收标准

- 所有起始日期/结束日期筛选均为手动输入框。
- 支持需求列出的全部输入格式。
- 成功输入后统一显示为 `YYYY/MM/DD`。
- 失败输入保留原文并出现明确错误提示。
- 无效字段不参与筛选。
- 起始日期晚于结束日期时，范围不生效并给出提示。
- 各页面筛选栏在桌面端与窄屏下均保持清晰、稳定、无明显布局问题。
