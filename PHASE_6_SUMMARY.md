# Phase 6: 首页概览升级与数据可视化增强 (Dashboard Enhancement)
**Date:** 2025-12-13

## 1. 目标 (Objectives)
全面升级首页概览 (Dashboard) 页面，使其更能反映年度经营状况，并提供更细致的财务和合同分类视图，同时保持与报表页面一致的视觉风格。

## 2. 完成事项 (Completed Tasks)

### 2.1 后端 API 升级 (`app/routers/dashboard.py`)
- **API 路径**: `/api/v1/dashboard/stats`
- **新增年度统计字段**:
    - `annual_upstream_count` / `annual_upstream_amount`: 本年度上游合同
    - `annual_receipts_amount`: 本年度实际回款
    - `annual_down_mgmt_count` / `annual_down_mgmt_amount`: 本年度下游及管理合同
    - `annual_payments_amount`: 本年度总支出 (下游+管理+费用)
- **图表数据增强**:
    - 收支趋势图: 增加详细的支出构成 (Downstream, Management, Non-contract) 以支持堆叠显示。
    - 饼图升级: 
        - 增加 `pie_company_data` (按公司分类)。
        - **重要**: 饼图数据 (Value) 统计口径从“合同数量”更改为“**签约总金额**”，更能反映业务权重。

### 2.2 前端页面重构 (`src/views/Dashboard.vue`)
- **统计卡片重设计**:
    - **内容更新**: 切换为年度统计数据。
    - **显示格式**: 增加“合同数量”显示，格式为 `X 单 / ¥ Y 万元`。
    - **排版调整**: 
        1. 年度上游合同
        2. 年度下游及管理合同 (调整至左侧)
        3. 年度回款
        4. 年度付款
    - **样式统一**: 全局卡片名称加粗，固定高度 180px。
    - **单位**: 统一使用“万元”，去小数取整 (Tooltip)。

- **收支趋势图升级**:
    - **布局**: 宽度调整为两卡片宽 (lg=12)。
    - **样式**: 复用报表页面的堆叠柱状图样式，清晰展示支出构成。
    - **标题与字体**: 标题加粗，以“本年度”为上下文。

- **双饼图布局实现**:
    - **布局**: 宽度调整为两卡片宽 (lg=12)，放置于趋势图右侧。
    - **内容**: 内部拆分为左右两个子图：
        - 左：按合同类别。
        - 右：按公司分类。
    - **交互**: 鼠标悬停显示“金额(万元) + 占比”，金额无小数展示。

### 2.3 视觉优化
- 全局卡片标题字体加粗 (`font-weight: bold`)。
- 优化了数值的显示字体大小 (20px) 以适应长文本。

## 3. 下一步计划 (Next Steps)
- 进行全站回归测试，确保数据准确性。
- 考虑增加更多交互维度（如快捷跳转到对应列表）。
