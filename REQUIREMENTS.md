# 项目需求文档：合同管理系统 (System Context Master File)

## 1. 项目概况
- **目标**: 开发一个局域网运行的工程合同管理系统，支持全生命周期管理（签订、应收/付、挂账、收/付款、结算）。
- **核心体验**:
    - 必须支持 PC 端宽屏管理和 **手机端 (Mobile)** 便捷访问（响应式布局）。
    - 强调数据的自动汇总与 Excel 互通。
- **文件约束**: 系统中所有上传文件（合同扫描件、发票、凭证、报告等）**强制限制为 .pdf 格式**，并支持前端在线弹窗预览。

## 2. 技术栈约束 (Tech Stack)
- **后端**: Python (FastAPI) + SQLAlchemy (Async, using asyncpg) + Pydantic.
- **数据库**: PostgreSQL.
- **前端**: Vue 3 (Script Setup) + Vite + Pinia + Element Plus (需处理移动端适配).
- **安全**: 基于 RBAC 的权限控制 (JWT Token).

## 3. 详细数据实体与字段定义 (Data Schema)

### 3.1 核心枚举 (Enums) - *必须严格遵循*
- **合同类别**: [总包合同, 专业分包, 劳务分包, 技术服务, 运营维护, 其他合同]
- **计价模式**: [总价包干, 单价包干, 工日单价, 费率下浮]
- **管理模式**: [自营, 合作, 挂靠]
- **应收/付款类别**: [预付款, 进度款, 完工款, 结算款, 质保金]
- **费用类别**: [公司费用, 项目费用]
- **费用分类**: [零星采购, 管理费, 培训费, 餐饮费, 交通费, 咨询费, 业务费, 租赁费, 资质费, 车辆使用费]
- **合同状态**: [执行中, 已完工, 已结算, 质保到期, 合同中止, 合同终止] (系统自动计算)

### 3.2 上游合同 (Upstream Contract) - *收入模型*
- **主表**: 合同序号 (UUID), 合同编号, 甲方单位, 乙方单位, 签约日期, 签约金额, 合同类别(Enum), 公司分类, 计价模式(Enum), 管理模式(Enum), 责任人, 合同文件路径(PDF Only), **状态 (Status)**.
- **子表1-应收款**: 应收款类别(Enum), 形成日期, 金额, 审批文件路径(PDF Only).
- **子表2-挂账**: 挂账日期, 金额, 发票号码, 发票文件路径(PDF Only).
- **子表3-收款**: 收款时间, 金额, 银行回单文件路径(PDF Only).
- **子表4-结算**: 结算编号, 结算金额, 结算日期(办结日期), 完工日期, 质保到期日期, 审核报告路径(PDF), 开工报告路径(PDF), 竣工报告路径(PDF), **状态(待审核/已审核)**.
- **逻辑**: 
    - 录入子表时，需自动回显主表的关键信息；系统需计算该合同的“累计应收”、“累计挂账”、“累计已收”、“累计结算”金额。
    - **状态自动计算逻辑**:
        1. 默认状态为"执行中"。
        2. 若存在结算记录且已填写完工日期，状态更新为"已完工"。
        3. 若结算金额 = 累计已收/付金额，状态更新为"已结算"。
        4. 若当前日期 > 质保到期日期，状态更新为"质保到期"。
        5. 可手动标记为"合同中止"或"合同终止"。

### 3.3 下游合同 (Downstream Contract) - *支出模型*
- **结构**: 基本同上游合同，但资金流向为“应付”和“付款”。
- **关键关联**: 必须包含外键关联上游合同 ID，并存储上游合同名称快照。
- **状态逻辑**: 同上游合同。

### 3.4 管理合同 (Management Contract)
- **结构**: 同下游合同。
- **特有子表-无合同费用报销**: 序号, 费用类别(Enum), 费用分类(Enum), 发生日期, 经办人, 关联上游合同ID, 负责人, 说明, 凭证文件路径(PDF Only).

## 4. 权限设计要求 (RBAC)
- 系统采用严格的 RBAC 模型。
- **后端 API 必须进行权限校验**。
- 前端按钮必须根据权限进行显隐控制。
- 具体角色和权限矩阵将在后续开发步骤中提供详细定义数据。

## 5. 首页概览 (Dashboard) & 报表统计
- **首页概览 (Dashboard)**:
    - **年度统计卡片**:
        - **年度上游合同签约总金额**: 显示本年度签订数量及总金额 (万元)。
        - **年度下游及管理合同签约总金额**: 显示本年度签订数量及总金额 (万元)。
        - **年度回款总金额**: 显示本年度实际回款总额 (万元)。
        - **年度付款总金额**: 显示本年度实际支付总额 (含下游、管理、无合同费用) (万元)。
    - **收支趋势图**:
        - 展示本年度月度收支情况。
        - 收入为绿色柱状图。
        - 支出为堆叠柱状图 (下游合同-红、管理合同-橙、无合同费用-灰)。
        - 宽度占据半屏 (lg=12)。
    - **合同分布饼图**:
        - 并排展示两个饼图：按**合同类别**分布、按**公司分类**分布。
        - 数据基础：基于**签约金额** (非数量) 计算占比。
        - 宽度占据半屏 (lg=12)。

- **报表统计页面 (Advanced Reports)**:
    - **目标**: 提供全维度的经营数据可视化看板，支持按年份和月份筛选。
    - **核心组件**:
        - **收支趋势图**: 同首页配置。
    - **应收/应付账款 (AR/AP)**: 
        - 重点展示“待收”与“待付”金额。
        - 包含总应收/已收、总应付/已付的进度条可视化（AR为蓝色/绿色，AP为橙色/绿色）。
    - **概况卡片**: 
        - 上游合同概况（按合同类别统计）。
        - 上游合同概况（按公司合同分类统计）。
        - 下游合同概况。
        - 管理合同概况。
        - 无合同费用概况。
- **多维饼图分析**:
    - **上游合同分类**: 展示不同合同类别的占比。
    - **上游合同公司分类**: 展示公司维度的合同占比。
    - **支出构成**: 展示各项支出（下游、管理、无合同）的资金构成。
    - **无合同费用分类**: 展示细分费用类型的分布。
- **交互与适配**:
    - 卡片标题加粗，关键数据醒目展示。
    - 图表提示框 (Tooltip) 限制在卡片范围内。
    - 完全适配移动端 (Mobile Responsive)，支持竖向堆叠布局。

## 6. 开发环境与运维 (DevOps & Troubleshooting)

### 6.1 一键启动环境 (Automated Environment Setup)
为解决开发环境启动繁琐、端口不一致及数据状态脏导致的问题，项目提供了一键启动脚本：
- **脚本位置**: `d:\LH_Contract_Docker\start_dev.ps1`
- **功能**:
    1.  **数据库检查**: 自动检测 PostgreSQL 容器状态，未启动则自动启动。
    2.  **后端环境检查**:
        - 校验 `backend\venv_win\Scripts\python.exe` 是否存在。
        - 若 Windows 后端虚拟环境缺失，脚本会直接报错退出，避免伪装成功。
    3.  **后端启动**: 在新窗口启动 FastAPI 服务 (Port 8000)。
    4.  **前端启动**: 在新窗口启动 Vite 开发服务 (Port 3000)。
    5.  **访问入口**: 启动完成后从 `http://localhost:3000` 访问前端。

### 6.2 常见问题解决 (Troubleshooting)
- **Problem**: 启动时提示找不到 `backend\venv_win\Scripts\python.exe`。
    - **Solution**: 先创建或恢复 Windows 开发虚拟环境，再重新运行 `start_dev.ps1`。
- **Problem**: Dashboard 显示为空，或出现 Network Error。
    - **Solution**: 检查 `frontend/vite.config.js` 中的 proxy target 是否为 `http://localhost:8000` (而非 9000)。`start_dev.ps1` 启动的后端默认为 8000 端口。
