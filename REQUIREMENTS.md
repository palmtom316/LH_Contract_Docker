# 项目需求文档：重庆蓝海电力合同管理系统 (System Context Master File)

## 1. 项目概况
- **目标**: 开发一个局域网运行的工程合同管理系统，支持全生命周期管理（签订、应收/付、挂账、收/付款、结算）。
- **核心体验**:
    - 必须支持 PC 端宽屏管理和 **手机端 (Mobile)** 便捷访问（响应式布局）。
    - 强调数据的自动汇总与 Excel 互通。
- **文件约束**: 系统中所有上传文件（合同扫描件、发票、凭证、报告等）**强制限制为 .pdf 格式**，并支持前端在线弹窗预览。

## 2. 技术栈约束 (Tech Stack)
- **后端**: Python (FastAPI) + SQLAlchemy (Async, using asyncpg) + Pydantic.
- **数据库**: PostgreSQL.
- **前端**: Vue 3 (Script Setup) + Vite + Pinia + Element Plus (需处理移动端适配).
- **安全**: 基于 RBAC 的权限控制 (JWT Token).

## 3. 详细数据实体与字段定义 (Data Schema)

### 3.1 核心枚举 (Enums) - *必须严格遵循*
- **合同类别**: [总包合同, 专业分包, 劳务分包, 技术服务, 运营维护, 其他合同]
- **计价模式**: [总价包干, 单价包干, 工日单价, 费率下浮]
- **管理模式**: [自营, 合作, 挂靠]
- **应收/付款类别**: [预付款, 进度款, 完工款, 结算款, 质保金]
- **费用类别**: [公司费用, 项目费用]
- **费用分类**: [零星采购, 管理费, 培训费, 餐饮费, 交通费, 咨询费, 业务费, 租赁费, 资质费, 车辆使用费]
- **合同状态**: [执行中, 已完工, 已结算, 质保到期, 合同中止, 合同终止] (系统自动计算)

### 3.2 上游合同 (Upstream Contract) - *收入模型*
- **主表**: 合同序号 (UUID), 合同编号, 甲方单位, 乙方单位, 签约日期, 签约金额, 合同类别(Enum), 公司分类, 计价模式(Enum), 管理模式(Enum), 责任人, 合同文件路径(PDF Only), **状态 (Status)**.
- **子表1-应收款**: 应收款类别(Enum), 形成日期, 金额, 审批文件路径(PDF Only).
- **子表2-挂账**: 挂账日期, 金额, 发票号码, 发票文件路径(PDF Only).
- **子表3-收款**: 收款时间, 金额, 银行回单文件路径(PDF Only).
- **子表4-结算**: 结算编号, 结算金额, 结算日期(办结日期), 完工日期, 质保到期日期, 审核报告路径(PDF), 开工报告路径(PDF), 竣工报告路径(PDF), **状态(待审核/已审核)**.
- **逻辑**: 
    - 录入子表时，需自动回显主表的关键信息；系统需计算该合同的“累计应收”、“累计挂账”、“累计已收”、“累计结算”金额。
    - **状态自动计算逻辑**:
        1. 默认状态为"执行中"。
        2. 若存在结算记录且已填写完工日期，状态更新为"已完工"。
        3. 若结算金额 = 累计已收/付金额，状态更新为"已结算"。
        4. 若当前日期 > 质保到期日期，状态更新为"质保到期"。
        5. 可手动标记为"合同中止"或"合同终止"。

### 3.3 下游合同 (Downstream Contract) - *支出模型*
- **结构**: 基本同上游合同，但资金流向为“应付”和“付款”。
- **关键关联**: 必须包含外键关联上游合同 ID，并存储上游合同名称快照。
- **状态逻辑**: 同上游合同。

### 3.4 管理合同 (Management Contract)
- **结构**: 同下游合同。
- **特有子表-无合同费用报销**: 序号, 费用类别(Enum), 费用分类(Enum), 发生日期, 经办人, 关联上游合同ID, 负责人, 说明, 凭证文件路径(PDF Only).

## 4. 权限设计要求 (RBAC)
- 系统采用严格的 RBAC 模型。
- **后端 API 必须进行权限校验**。
- 前端按钮必须根据权限进行显隐控制。
- 具体角色和权限矩阵将在后续开发步骤中提供详细定义数据。
