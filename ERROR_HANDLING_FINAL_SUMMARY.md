## 💡 最佳实践总结

### 1. 错误类型选择指南

```python
# 404 - 资源不存在
raise ResourceNotFoundError("资源类型", id)

# 400 - 数据验证失败
raise ValidationError("消息", field_errors)

# 401 - 认证失败
raise AuthenticationError("消息", detail)

# 403 - 权限不足
raise PermissionDeniedError("消息", detail)

# 409 - 记录重复
raise DuplicateRecordError("类型", "字段", "值")

# 500 - 数据库/系统错误
raise DatabaseError("消息", detail)
```

### 2. 错误消息编写规范

**结构**:
- **message**: 简短、用户友好
- **detail**: 详细、技术性
- **data**: 结构化附加信息

**示例**:
```python
raise ValidationError(
    message="数据验证失败",  # 给用户看
    field_errors={           # 给开发者看
        "amount": "金额必须大于0",
        "date": "日期格式错误"
    }
)
```

### 3. 一致性原则

**所有类似操作使用相同模式**:
```python
# 所有get操作
if not record:
    raise ResourceNotFoundError(type, id)

# 所有ID验证
if path_id != body_id:
    raise ValidationError("ID不匹配", errors)
```
