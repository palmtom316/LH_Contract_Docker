## 二、新发现问题及优化建议（12项）

### 🔴 高优先级 (2项)

#### 问题 1: `reports.py` 代码体积过大
**文件**: `backend/app/routers/reports.py`  
**问题**: 单个文件 1569 行、61KB，包含 18 个端点，职责过重
**风险**: 增加维护难度，测试困难
**建议**: 拆分为多个模块

```
routers/
├── reports/
│   ├── __init__.py          # 聚合路由
│   ├── summary.py           # 合同汇总统计
│   ├── finance_trend.py     # 财务趋势
│   ├── expense_breakdown.py # 费用明细
│   └── exports.py           # Excel 导出
```

---

#### 问题 2: 前端组件缺少 TypeScript 类型保护
**问题**: 整个前端使用 JavaScript，缺乏类型安全
**风险**: 大型项目重构困难，IDE 提示不完善
**建议**: 逐步迁移关键模块

```typescript
// 建议为核心 API 模块添加类型 (types/api.d.ts)
export interface ContractUpstream {
  id: number;
  serial_number: number;
  contract_name: string;
  contract_amount: number;
  status: '执行中' | '已完成' | '已结算' | '已终止';
}
```

---

### 🟡 中优先级 (6项)

#### 问题 3: 依赖版本滞后
**问题**: 多个核心依赖需要更新

| 包 | 当前版本 | 最新版本 | 建议 |
|---|---|---|---|
| FastAPI | 0.109.x | 0.115.x | 更新 |
| SQLAlchemy | 2.0.25 | 2.0.36 | 更新 |
| Vue | 3.4.x | 3.5.x | 更新 |
| Element Plus | 2.5.x | 2.9.x | 更新 |

**建议**: 创建依赖更新分支，逐步测试升级

---

#### 问题 4: 数据库查询 N+1 问题风险
**发现**: 多处合同列表查询使用 `selectinload`，但部分子查询仍可能触发 N+1
**建议**: 添加查询性能监控，使用 `joinedload` 优化热点查询

```python
# 优化示例
stmt = select(ContractUpstream).options(
    joinedload(ContractUpstream.receivables),
    joinedload(ContractUpstream.settlements)
).where(...)
```

---

#### 问题 5: 前端缺少全局错误上报
**问题**: 生产环境错误难以追踪
**建议**: 集成错误监控服务（可选 Sentry 或自建）

```javascript
// main.js
app.config.errorHandler = (err, vm, info) => {
  console.error('Global error:', err, info);
  // 建议上报到日志服务
};
```

---

#### 问题 6: 内存缓存无过期机制
**文件**: `backend/app/core/cache.py`
**问题**: 内存缓存 fallback 模式无 TTL 支持，可能导致内存泄漏
**建议**: 使用 `cachetools` 库实现带过期的内存缓存

```python
from cachetools import TTLCache
self.memory_cache = TTLCache(maxsize=1000, ttl=300)
```

---

#### 问题 7: 移动端表格体验欠佳
**发现**: 大型数据表格在移动端水平滚动体验不佳
**建议**: 为移动端提供卡片视图替代表格视图

---

#### 问题 8: PDF 预览依赖外部服务
**发现**: 部分 PDF 使用 Google Docs 预览
**风险**: 外部服务依赖，隐私和可用性风险
**建议**: 使用本地 PDF.js 渲染

---

### 🟢 低优先级 (4项)

#### 问题 9: 前端 Console 日志残留
**问题**: 生产代码包含 `console.log` 调试语句
**建议**: 配置 ESLint 规则禁止提交 console 语句

```javascript
// .eslintrc.js
rules: {
  'no-console': process.env.NODE_ENV === 'production' ? 'error' : 'warn'
}
```
