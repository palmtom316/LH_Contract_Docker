## 🎯 部署方案对比

### 方式一：Docker Compose部署
- **适用场景**: 已有Linux服务器
- **复杂度**: 中等
- **灵活性**: 高
- **资源隔离**: 容器级别

### 方式二：PVE LXC部署（推荐）⭐
- **适用场景**: 裸机服务器（如铭凡MS-A2）
- **复杂度**: 较高（但有自动脚本）
- **灵活性**: 非常高
- **资源隔离**: 虚拟化级别
- **优势**:
  - ✅ 完整的虚拟化隔离
  - ✅ 资源利用率高
  - ✅ 便于快照和备份
  - ✅ 可运行多个容器
  - ✅ 专业的管理界面

## 🚀 部署流程（使用自动脚本）

### 第一步：安装PVE系统
```bash
# 1. 下载Proxmox VE ISO
# 2. 制作U盘启动盘
# 3. 安装到服务器（IP: 192.168.72.100）
# 4. 访问: https://192.168.72.100:8006
```

### 第二步：创建LXC容器
```bash
# 通过PVE Web界面创建容器
# - CT ID: 100
# - 主机名: lh-contract
# - IP: 192.168.72.101/24
# - CPU: 4核, 内存: 8GB, 磁盘: 50GB
```

### 第三步：自动部署应用
```bash
# SSH登录到容器
ssh root@192.168.72.101

# 下载并执行自动部署脚本
wget https://raw.githubusercontent.com/palmtom316/LH_Contract_Docker/release/v1.1/scripts/deploy_lxc.sh
chmod +x deploy_lxc.sh
sudo ./deploy_lxc.sh
```

### 第四步：访问应用
```
前端: http://192.168.72.101:3000
后端: http://192.168.72.101:8000
API文档: http://192.168.72.101:8000/docs
```

**总耗时**: 约30-60分钟（取决于网络速度和硬件性能）

## 🔧 运维命令快查

### 应用管理
```bash
cd /opt/lh-contract

# 查看状态
docker compose -f docker-compose.prod.yml ps

# 查看日志
docker compose -f docker-compose.prod.yml logs -f

# 重启服务
docker compose -f docker-compose.prod.yml restart

# 停止/启动
docker compose -f docker-compose.prod.yml down
docker compose -f docker-compose.prod.yml up -d
```

### 容器管理（PVE主机）
```bash
pct start 100      # 启动
pct stop 100       # 停止
pct restart 100    # 重启
pct enter 100      # 进入
```
